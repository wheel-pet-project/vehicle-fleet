# Микросервис VehicleFleet

Микросервис управления автопарком для пет-проекта Wheel

## Описание

Микросервис VehicleFleet предоставляет полный набор функций для управления автопарком в ИС для каршеринга. Сервис построен на основе Clean Architecture с использованием Domain-Driven Design (DDD) принципов и включает управление транспортными средствами, моделями автомобилей и их жизненным циклом.

## Архитектура

Проект следует принципам Clean Architecture и состоит из следующих слоев:

- **Api** - gRPC API слой с контрактами и мапперами
- **Application** - use cases, команды, запросы и обработчики доменных событий
- **Domain** - доменная логика, агрегаты и саги
- **Infrastructure** - реализация репозиториев и внешних сервисов

## Основные функции

### 🚗 Управление транспортными средствами
- **Добавление автомобиля** - регистрация нового транспортного средства в автопарке
- **Удаление автомобиля** - исключение автомобиля из автопарка
- **Поиск автомобилей** - получение списка автомобилей с фильтрацией по статусу
- **Детальная информация** - получение полной информации об автомобиле

### 📍 Геолокация и поиск
- **Поиск в квадрате** - поиск автомобилей в заданной географической области
- **Отслеживание местоположения** - получение текущих координат автомобиля
- **Фильтрация по статусу** - поиск автомобилей по состоянию (доступен, занят, на обслуживании)

### 🔄 Жизненный цикл автомобиля
- **Добавление автомобиля** - добавление автомобиля в автопарк
- **Добавлен** - автомобиль успешно добавлен в автопарк
- **Не добавлен** - автомобиль не добавлен по причине сбоя саги 
- **Подготовка к выпуску** - маркировка автомобиля как готового к аренде
- **Выпуск в аренду** - перевод автомобиля в статус доступного
- **Занятие автомобиля** - перевод автомобиля в статус занятого
- **Отправка на обслуживание** - перевод автомобиля в статус обслуживания
- **Удален** - удаление автомобиля из автопарка


### 🏷️ Управление моделями автомобилей
- **Добавление модели** - создание новой модели автомобиля с тарифами
- **Обновление категории** - изменение категории модели
- **Обновление тарифов** - изменение цен за минуту, час или день
- **Получение моделей** - просмотр всех доступных моделей

## Технологический стек

- **.NET 8** - основная платформа
- **gRPC** - API протокол
- **PostgreSQL** - основная база данных
- **Kafka** - обмен сообщениями
- **MediatR** - паттерн Mediator для use cases
- **FluentResults** - обработка результатов операций
- **Entity Framework Core** - ORM для работы с базой данных

## API Endpoints

Сервис предоставляет следующие gRPC методы:

### Модели автомобилей
- `AddModel` - создание новой модели автомобиля
- `UpdateModelCategory` - обновление категории модели
- `UpdateModelTariff` - обновление тарифов модели
- `GetAllModels` - получение всех моделей
- `GetModelById` - получение модели по ID

### Транспортные средства
- `AddVehicle` - добавление нового автомобиля
- `MarkAsReadiedForReleaseVehicle` - подготовка автомобиля к выпуску
- `ReleaseVehicle` - выпуск автомобиля в аренду
- `SendToServiceVehicle` - отправка автомобиля на обслуживание
- `DeleteVehicle` - удаление автомобиля
- `GetAllVehicles` - получение всех автомобилей с фильтрацией
- `GetVehiclesInSquare` - поиск автомобилей в географической области
- `GetVehicleById` - получение информации об автомобиле
- `GetVehicleDetailsById` - получение детальной информации об автомобиле

## Доменные события

Система генерирует следующие доменные события:
- `VehicleAddedDomainEvent` - при добавлении автомобиля
- `VehicleDeletedDomainEvent` - при удалении автомобиля
- `VehicleReadiedForReleaseDomainEvent` - при подготовке к выпуску
- `VehicleReleasedDomainEvent` - при выпуске автомобиля
- `VehicleOccupyingProcessedDomainEvent` - при обработке занятия
- `VehicleServicedDomainEvent` - при отправке на обслуживание

## Сага добавления автомобиля

Система включает сагу для управления процессом добавления автомобиля:
- **VehicleAddingSaga** - координация процесса добавления
- **VehicleAddingState** - состояния процесса
- **VehicleAddingSagaEvent** - события саги
- **VehicleAddingProcess** - логика обработки процесса

│   └── SharedKernel/     # Общие доменные концепции
├── Infrastructure/       # Инфраструктурный слой
└── Tests/               # Тесты
    ├── UnitTests/       # Модульные тесты
    └── IntegrationTests/ # Интеграционные тесты
